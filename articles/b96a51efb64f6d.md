# Upgrading Using AI

もし Laravel Boost でアップグレードを自動化したいときは、 `/upgrade-laravel-13` のコマンドを Agent になげればやってくれるみたいです。

## [High] Updating Dependency , [High] Updating Laravel Installer

composer.json でアップグレードが必要なパッケージは四つあります。

- laravel/framework to ^13.0
- laravel/tinker to ^3.0
- phpunit/phpunit to ^12.0
- pestphp/pest to ^4.0

もし Laravel installer を使っているときは、それも `composer global update laravel/installer` でアップグレードしてください。

## [High][New][Upgrade] Request Forgery Protection

Laravel CSRF Middleware （ **VerifyCsrfToken** ）の名前が変わり、Request Forgery Protection( **PreventRequestForgery** )になりました。

[**Sec-Fetch-Site** header -- developer.mozilla.org](https://developer.mozilla.org/ja/docs/Web/HTTP/Reference/Headers/Sec-Fetch-Site) を使うようになります。


```php
use Illuminate\Foundation\Http\Middleware\PreventRequestForgery;
use Illuminate\Foundation\Http\Middleware\VerifyCsrfToken;
 
// Laravel <= 12.x
->withoutMiddleware([VerifyCsrfToken::class]);
 
// Laravel >= 13.x
->withoutMiddleware([PreventRequestForgery::class]);
```

## [New] Queue Routing

Queue Facade に route がつきました。Job内で connection や queue を指定しない方法で使えるようになります。

## [New] Expanded PHP Attributes

PHP8 についた [Attribute -- www.php.net](https://www.php.net/manual/ja/language.attributes.overview.php) に対応しました（12の頃からしてなかったっけ？）。QueueJobにはありましたが、Controllerにも対応ってことかしら。

```php
<?php
 
namespace App\Http\Controllers;
 
use App\Models\Comment;
use App\Models\Post;
use Illuminate\Routing\Attributes\Controllers\Authorize;
use Illuminate\Routing\Attributes\Controllers\Middleware;
 
#[Middleware('auth')]
class CommentController
{
    #[Middleware('subscribed')]
    #[Authorize('create', [Comment::class, 'post'])]
    public function store(Post $post)
    {
        // ...
    }
}
```


## [New] Cache TTL Extension

`Cache::touch($key, $second)` が追加されました。これはキャッシュの有効期限を延長します。

## [New] Semantic / Vector Search

Vector 検索などセマンティックな検索に対応しました。この機能は AI SDK などに広く使われ、それらに説明されています。

```php
$documents = DB::table('documents')
    ->whereVectorSimilarTo('embedding', 'Best wineries in Napa Valley')
    ->limit(10)
    ->get();
```

## [Low] Cache Prefixes and Session Cookie Names

Laravel 標準だった session name などの規則が変更されました。だいたいの人はここをアプリケーションに準じた名前に変更しているので関係がありませんし、追従して変更する必要もないと思われます。

```php
// Laravel <= 12.x
Str::slug((string) env('APP_NAME', 'laravel'), '_').'_cache_';
Str::slug((string) env('APP_NAME', 'laravel'), '_').'_database_';
Str::slug((string) env('APP_NAME', 'laravel'), '_').'_session';
 
// Laravel >= 13.x
Str::slug((string) env('APP_NAME', 'laravel')).'-cache-';
Str::slug((string) env('APP_NAME', 'laravel')).'-database-';
Str::snake((string) env('APP_NAME', 'laravel')).'_session';
```

## [Low] Collection Model Serialization Restores Eager-Loaded Relations

よくわからん。

> When Eloquent model collections are serialized and restored (such as in queued jobs), eager-loaded relations are now restored for the collection's models.

Eloquentのモデルコレクションをシリアライズして復元（キューに入れられたジョブなど）する際、以前は欠落していたコレクションのモデルに対して、Eagerロードされたリレーションが復元されるようになりました。

> If your code depended on relations not being present after deserialization, you may need to adjust that logic.

もし、デシリアライズ後にリレーションが存在しないことに依存するコードを記述していた場合は、そのロジックを調整する必要があるかもしれません。

多分 `Job::detach(DoSomethingClass(User::with('items')->where('join_date','>', now()->subDay())->get() );` 的なコードで **DoSomethingClass** に渡された **$user->items** の items の保証があるよ的な…？メモリーとか datastore に乗らん気がするけど…

## [Low] Container::call and Nullable Class Defaults

**Container::call** で引数に型指定した時に、任意のインスタンスを渡してたのを、必ずnullを渡すようになったみたいです。

## [Low] Domain Route Registration Precedence

Domain有り→無しと順番に記述しないとうまくマッチングしなかった（Routing の登録順番に依存していた）のが、ドメイン優先となりました。という感じっぽい。

> Routes with an explicit domain are now prioritized before non-domain routes in route matching.

ドメインが明示されているルートは、ルートマッチングにおいて、ドメインがないルートよりも優先されるようになりました。

> This allows catch-all subdomain routes to behave consistently even when non-domain routes are registered earlier. If your application relied on previous registration precedence between domain and non-domain routes, review route matching behavior.

これにより、キャッチオールサブドメインルートは、ドメインがないルートが先に登録されていたとしても、一貫した動作をするようになります。もし、アプリケーションがドメインありのルートとドメインなしのルート間の以前の登録順序に依存していた場合、ルートマッチングの動作を見直してください。

## [Low] JobAttempted Event Exception Payload

**Illuminate\Queue\Events\JobAttempted** の **exceptionOccurred** プロパティ名が **exception** に変更されました。もし使用している場合は、変更をお願いします。

## [Low] Manager extend Callback Binding

Cache Driver などを統括している Manager クラスから、Custom Driver を登録して extend メソッドを使っている時の話です。extend で callback を渡した時にManagerに bindされるので **$this** が変わるので、 **use(...)** でキャプチャしてあげる必要がある。

> Custom driver closures registered via manager extend methods are now bound to the manager instance.
> If you previously relied on another bound object (such as a service provider instance) as $this inside these callbacks, you should move those values into closure captures using use (...).

## [Low] MySQL DELETE Queries With JOIN, ORDER BY, and LIMIT

MySQL の `Delete ... Join` に対応し、**ORDER BY**, **LIMIT** に対応をしました。

## [Low] Pagination Bootstrap View Names

Pagination のテンプレートビューが変わりました。

```
// Laravel <= 12.x
pagination::default
pagination::simple-default
// Laravel >= 13.x
pagination::bootstrap-3
pagination::simple-bootstrap-3
```

## [Low] Polymorphic Pivot Table Name Generation

> When table names are inferred for polymorphic pivot models using custom pivot model classes, Laravel now generates pluralized names.

> If your application depended on the previous singular inferred names for morph pivot tables and used custom pivot classes, you should explicitly define the table name on your pivot model.


## [Low] QueueBusy Event Property Rename

> The Illuminate\Queue\Events\QueueBusy event property $connection has been renamed to $connectionName for consistency with other queue events.

> If your listeners reference $connection, update them to $connectionName.

### [Low] Queue Contract Method Additions

> The Illuminate\Contracts\Queue\Queue contract now includes queue size inspection methods that were previously only declared in docblocks.
> If you maintain custom queue driver implementations of this contract, add implementations for:

- pendingSize
- delayedSize
- reservedSize
- creationTimeOfOldestPendingJob


## [Low] Str Factories Reset Between Tests

Str factories を test で使っているときに、都度リセットされます。UUIDやULID,random string に依存している場合、ちょっとテストに影響があるかもしれません。

> Laravel now resets custom Str factories during test teardown.
> If your tests depended on custom UUID / ULID / random string factories persisting between test methods, you should set them in each relevant test or setup hook.

### [Low] Js::from Uses Unescaped Unicode By Default

> Illuminate\Support\Js::from now uses JSON_UNESCAPED_UNICODE by default.
> If your tests or frontend output comparisons depended on escaped Unicode sequences (for example \u00e8), update your expectations.
